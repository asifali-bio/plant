# Contributing

Thank you for your interest in contributing to this project.

## How to Contribute

1. Fork the repository
2. Create a new branch (`feature/your-feature` or `fix/your-fix`)
3. Make your changes
4. Submit a pull request with a clear description of your changes

## Guidelines

- Keep changes focused and minimal
- Write clear, readable, and well-documented code
- Follow existing naming conventions and structure
- Where applicable, ensure reproducibility of analyses

## Issues

- Use issues to report bugs or suggest features
- Provide as much detail as possible (inputs, outputs, errors, context)

## Notes

This project emphasizes clarity, reproducibility, and methodological consistency.  
Contributions should align with these principles.

## Code of Conduct

By participating, you agree to abide by this project's Code of Conduct.
